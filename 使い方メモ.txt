★githubにログイン
ssh -T git@github.com
★登録しないファイルを管理
.gitと同じフォルダい.gitignoreファイルを作成してコミット対象外のファイルパスを記述
★ステージング登録前のバージョンに戻す
git checkout ファイル名
★コミットを取り消す
・メッセージを元に取り消し(コミットログ上も取り消す)
git reset --hard コミットした際のメッセージ(正規表現可能)
・コミットIDを元に取り消し(コミットログに取り消しを追加)
git revert コミットID
★ブランチの切り替え
git checkout -b feature/create-xx
git branch
★ブランチをプッシュ
git push origin feature/create-xx
★ブランチをマージ
・GitHubの画面から対象ブランチにプルリクエストを作成
・そのあとにMerge Pull requestを行う
★リモートの変更を取り込む
git pull


①リポジトリ作成
以下コマンドでローカルにリポジトリを作成うる
git init

②バージョン登録をする
・状態を確認する
git status
・ステージングに登録する
git add *
・リポジトリにコミットする
git commit -m "メッセージ"
・コミット履歴を確認する
git log
・変更の内容を確認する
git show

③ローカルの変更の差分を確認する
git diff

④ステージングの変更の差分を確認する
git diff --cached

⑤リモートリポジトリを新規作成する
git remote add origin git@github.com:yushin1402/html.git
・originに紐づいているURLの確認
git config remote.origin.utl
・originに紐づいているURLの変更
git remote set-url origin XXXXXXXX


⑥リモートリポジトリにコミットを登録する
git push -u origin master
